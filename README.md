# 🌐 AstrBot 局域网 WOL 唤醒插件

简易、高效的局域网唤醒工具，支持单播/广播双发、动态开机状态检测。

## ✨ 特性

- **单播+广播双发机制**：底层自动并发发送唤醒包，有效解决 Docker 桥接模式下广播隔离问题，大幅提升唤醒成功率。
- **动态开机检测**：发送唤醒包后自动轮询状态，实时反馈等待进度，完美适配慢速启动设备（如 AMD 平台较长自检时间）。
- **安全隔离**：严格限制仅允许指定白名单用户在私聊中使用，防止群聊滥用。

## 🐳 Docker 部署注意事项

1. **最简方案 (Host模式)**：推荐将 Docker 容器网络模式设置为 `host`，此时无视所有网络隔离，即插即用。
2. **桥接模式**：
   - 默认情况下，Docker 桥接网络的广播包（255.255.255.255）无法穿透到宿主机物理局域网。
   - **本插件已做优化**：只要在配置中填写了目标电脑的 `ip`，插件会自动采用“单播+广播”双发策略。
   - **进阶技巧**：建议在路由器（如 OpenWrt）中为目标电脑设置 **静态 ARP 绑定**。这样即使电脑处于关机状态，单播包也能精准直达目标网卡，实现 100% 唤醒成功率。

## 📋 指令说明

- `/绑定 <MAC地址> [IP地址]` 
  - 绑定需要唤醒的设备（强烈建议同时填写 IP 地址）。
  - 示例：`/绑定 AA:BB:CC:DD:EE:FF 192.168.1.100`
- `/开机` 
  - 发送 WOL 唤醒包，自动等待并最多检测 3 次状态（每次间隔 30 秒），直到设备上线或检测结束。
- `/我的电脑` 
  - 立即查询当前设备在线状态。
- `/局域网唤醒帮助` 
  - 显示完整帮助信息。

## ⚙️ 配置项说明

| 配置项 | 说明 |
| --- | --- |
| `mac` | 绑定的目标设备 MAC 地址。 |
| `ip` | 绑定的目标设备 IP 地址。**必填**：用于单播唤醒和 Ping 状态检测，未配置时无法自动查看状态。 |
| `broadcast` | 广播地址（默认 `255.255.255.255`），使用单播时可忽略此项。 |
| `port` | 唤醒包发送端口（默认 `9`）。 |
| `allowed_users` | 允许使用插件的用户 ID 列表。**必填**：留空将导致所有指令失效（仅限私聊环境生效）。 |

## 🛠️ 安装方法

1. 将本插件文件夹放入 AstrBot 的 `data/plugins/` 目录下。
2. 重启 AstrBot，或在后台执行 `reload` 重载插件。
3. 插件会根据 `requirements.txt` 自动安装所需依赖（`ping3`）。若自动安装失败，请手动执行：`pip install ping3`。
4. **重要**：安装后必须进入 AstrBot 插件管理界面，填写 `allowed_users` 白名单和设备信息，否则无法使用任何功能。
